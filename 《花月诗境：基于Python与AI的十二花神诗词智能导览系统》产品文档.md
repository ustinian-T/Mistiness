# 一、引言

## 1.1 编写目的

本文档完整定义《花月诗境》智能导览系统的产品需求、技术实现、数据规范、AI 协作流程、交付标准，统一开发、演示、报告撰写标准；同时满足课程作业硬性要求：可运行程序、数据处理、AI 工具应用、过程记录、讲解视频、运行截图，规范 AI 使用说明、留存提示词与人工修改记录。

## 1.2 项目背景

2026 年央视春晚创意节目《贺花神》将十二月令花卉、古典诗词、历代文人意象结合，以数字化舞台形式传播传统月令花神文化。现有传播形式多为 PPT 图文资料展示，缺少交互式数据检索、智能匹配、自动化文化解析能力。
本项目以 Node.js 为基础，融合数据处理、AI 辅助创作、可视化统计、语音合成，打造轻量化传统文化交互 Web 应用，实现**传统文化数字化、人机协同学习**，区别于纯文科资料展示类作业，充分体现计算思维、分支循环函数、数据结构、人工智能工具应用。

## 1.3 术语定义

1. 十二花神：对应 1-12 月月令花卉，匹配专属文人、经典诗词、朝代文化意象
2. Express：Node.js 轻量化 Web 框架，用于构建 RESTful API 服务
3. CSV：结构化数据表，存储十二花神全量基础数据
4. AI 协同：AI 仅作为初稿生成工具，所有内容人工核查、修正、定稿，非完全替代人工
5. 数字人播报：使用第三方 AI 数字人工具，导入程序生成的讲解台词生成短视频片段
6. 关键词匹配：基于字典映射 + 模糊评分 + AI 语义的三层花卉智能推荐算法
7. TTS：Text-to-Speech，文本转语音合成技术
8. SSE：Server-Sent Events，服务器向浏览器流式推送数据的技术
9. Glassmorphism：液态玻璃设计风格，半透明毛玻璃效果的 UI 设计语言

2. 2026 央视春晚《贺花神》节目文化素材

# 二、产品概述与定位

## 2.1 产品全称

《花月诗境：基于 Node.js 与 AI 的十二花神诗词智能导览系统》

## 2.2 一句话定位

基于 Node.js 开发的交互式传统文化学习 Web 应用，用户通过月份/情感关键词查询十二花神文化数据，系统自动输出花卉、诗人、朝代、经典诗句、文化意象、AI 人工修正讲解稿，附带朝代诗词数据可视化统计，支持生成播客文案与 TTS 语音合成，完整覆盖编程作业全部考核要点。

## 2.3 核心产品价值

1. **技术价值**：完整落地变量、分支、循环、自定义函数、CSV 数据读写、数据统计、可视化图表，满足程序设计考核要求；前后端分离架构，RESTful API 设计，体现软件工程实践。
2. **文化价值**：以春晚热点为切入点，系统化梳理十二月令花神与古典诗词体系；
3. **AI 实践价值**：标准化人机协同流程，完整记录 AI 生成、人工校验、修改迭代全过程，符合课程 AI 应用作业规范；支持大模型实时调用（播客生成、语义匹配、诗词推荐、语音合成）；
4. **展示价值**：液态玻璃网页界面 + 数据图表 + 语音播客 + 数字人短片 + 完整演示视频，课堂展示效果优于控制台程序与静态 PPT。

## 2.4 目标用户画像

1. 课程评审教师：考核编程能力、数据处理、AI 合理使用、项目完整性；
2. 学生使用者：传统文化爱好者，通过月份、情感关键词快速学习月令诗词；
3. 演示受众：课堂观众，可直观查看程序交互、数据分析、AI 讲解成果。

## 2.5 产品核心差异化

摒弃传统纯图文 PPT 展示方案，不做高成本、难落地的全文生视频；采用**轻量化 Node.js Web 应用 + 本地 CSV 数据 + AI 辅助文案 + TTS 语音合成 + 低成本数字人片段**组合，低成本、易部署、代码可完整运行、技术亮点充足。

# 三、需求分析

## 3.1 功能需求

### 基础核心功能（必做）

1. 十二花神按月查询功能：下拉选择/点击月份，输出花卉、花神、诗人、朝代、诗句、文化意象、讲解稿，配花卉图与人物画像；
2. 关键词智能推荐功能：输入意象关键词，自动匹配对应花卉，支持标签快捷输入；
3. 朝代数据统计可视化：统计各朝代诗人数量，生成柱状统计图 + 数据明细表；
4. 查询结果本地保存：将用户查询记录导出至 TXT 文件；
5. 异常输入容错处理：非法月份、无效关键词给出友好提示。

### AI 辅助增值功能（作业加分项）

1. AI 生成标准化诗词讲解稿，人工修正后存入数据库；
2. 生成适配数字人朗读的简短播报台词；
3. AI 单人播客文案生成（流式输出，约 500 字）；
4. AI 双人对话播客生成（男女主持人，约 800-1000 字）；
5. TTS 语音合成（单人/双人对话双模式，在线播放与下载）；
6. AI 扩展诗词推荐（每花 3-5 首，含赏析）；
7. AI 语义关键词匹配（语义级推荐，含匹配理由）；
8. 完整留存 AI 提示词、生成初稿、人工修改记录，用于报告撰写。

### 展示配套功能

1. 液态玻璃（Glassmorphism）UI 设计，浅蓝诗境主题；
2. 12 月花卉高清图 + 12 花神人物画像；
3. 渐入动画、悬停动效、平滑滚动；
4. 导出讲解文本，对接剪映/腾讯智影等工具生成数字人短视频；
5. 系统运行截图、录屏导出，用于提交材料。

## 3.2 非功能需求

1. **易用性**：界面简洁，中文交互，无需专业编程基础即可操作；
2. **可运行性**：Windows/Mac 通用，仅需安装 Node.js 基础依赖；
3. **可维护性**：数据独立存储于 CSV，新增花卉、诗句无需修改主代码；前后端分离，模块清晰；
4. **可展示性**：网页界面布局清晰，图表适配课堂投屏；
5. **可追溯性**：所有 AI 生成内容、修改过程、提示词完整留存，有据可查；
6. **轻量化**：核心功能无需 AI Key，本地零成本运行；AI 功能可选增强，按需启用。

## 3.3 项目约束条件

1. 技术约束：后端 Node.js + Express，前端原生 HTML/CSS/JS，可视化 Chart.js，AI 调用 DashScope API；
2. 成本约束：基础功能零成本，AI 功能使用免费额度，数字人采用免费在线工具；
3. 内容约束：诗词、诗人、朝代史料必须人工核验，禁止直接使用 AI 错误史料；
4. 交付约束：必须同时提交源代码、数据表、报告、视频、AI 截图、运行录屏；
5. 时间约束：整体开发周期适配课程作业周期，7 天内完成全流程。

## 3.4 不包含功能（边界说明）

1. 不开发移动端 APP、小程序（响应式网页适配移动端即可）；
2. 不搭建后端数据库服务（仅本地 CSV 文件存储）；
3. 不做完整长篇数字人影片，仅 1 分钟精简播报片段；
4. 不拓展二十四节气、百花全量数据，仅聚焦十二花神核心数据。

# 四、系统总体设计

## 4.1 技术架构（三层分层设计）

1. **数据层**：本地 flower_poetry_data.csv 结构化数据表，存储十二花神全部基础数据；配套花卉图片、花神画像资源。
2. **业务逻辑层**：Node.js + Express 后端服务，封装 FlowerService（查询/推荐/统计/保存）、AIService（大模型调用/TTS 合成）；提供 RESTful API 接口。
3. **展示交互层**：原生 HTML/CSS/JS 前端，液态玻璃 UI，Chart.js 图表，提供导航、选择框、输入框、图表展示、文本输出、音频播放界面。

## 4.2 系统整体流程图

```Plain Text
程序启动 → 读取CSV十二花神数据库 → 启动 Express 服务 → 加载前端页面
    ↓
用户在浏览器选择功能分支：
├─ 功能1：按月查询 → 选择月份 → 调用 /api/flowers/month/:month → 输出完整花神信息+AI讲解稿+图片
├─ 功能2：关键词推荐 → 输入意象词 → 本地匹配 / AI 语义匹配 → 推荐对应花卉
├─ 功能3：朝代统计分析 → 调用 /api/stats/dynasty → Chart.js 生成柱状图+明细表
├─ 功能4：播客生成 → 选择月份 → AI 生成单人/双人播客 → TTS 语音合成 → 在线播放
└─ 功能5：AI 诗词扩展 → 点击查询更多 → AI 推荐 3-5 首相关诗词
    ↓
可选：保存本次查询记录至本地 record.txt
    ↓
继续浏览 / 关闭页面
```

# 六、核心功能模块详细说明

## 模块 1：十二花神按月查询模块

1. **输入**：用户下拉选择 1-12 月，或从首页月历卡片点击跳转
2. **处理逻辑**：后端 csv-parse 读取 CSV，匹配 month 字段，提取该行全部数据
3. **输出内容**：月份、花卉、花神、诗人朝代、经典诗句、文化意象、朝代诗词风格、AI 讲解文案、花卉图片、花神画像、百度百科跳转链接
4. **异常处理**：输入数字 > 12 或 < 1，返回 400 错误
5. **扩展按钮**：AI 查询更多诗词、生成播客文案、保存查询记录

## 模块 2：关键词智能花卉推荐模块

1. **内置关键词映射字典**（38 个关键词）

```Plain Text
KEYWORD_MAP = {
    高洁: ["梅花", "荷花", "菊花"],
    爱情: ["桃花", "杏花"],
    隐逸: ["菊花"],
    团圆: ["桂花"],
    夏日: ["荷花"],
    清冷: ["梅花"],
    孤傲: ["梅花"],
    春日: ["桃花", "杏花"],
    温柔: ["杏花"],
    明艳: ["桃花"],
    ...（共 38 个）
}
```

2. **输入**：用户文本框输入意象关键词，或点击预设关键词标签
3. **处理逻辑**：
   - 第一层：本地精确字典匹配
   - 第二层：本地模糊匹配（字符级评分排序）
   - 第三层（有 AI Key）：大模型语义匹配，优先返回
4. **输出**：推荐花卉名称 + 对应月份 + 花神人物 + 朝代 + 匹配理由
5. **容错**：无匹配关键词时提示 "暂无对应花卉，请更换关键词"

## 模块 3：朝代诗词数据统计可视化模块

1. **处理逻辑**：后端对 dynasty 字段分组计数，统计每个朝代对应花神数量
2. **可视化**：Chart.js 生成柱状图，横轴朝代、纵轴文人数量，渐变蓝色，圆角柱状，液态玻璃容器
3. **输出**：网页内统计图 + 数据明细表格，可截图保存用于作业材料

## 模块 4：查询结果保存模块

1. **触发方式**：用户点击 "保存查询记录" 按钮
2. **处理逻辑**：读取当前页面查询数据，追加写入 record.txt
3. **记录格式**：查询时间 + 查询类型 + 查询输入 + 查询结果，分隔线分隔
4. **用途**：留存交互记录，作为程序数据处理功能佐证

## 模块 5：数字人台词生成模块

1. **处理逻辑**：截取 ai_script 字段，精简调整至 80 字以内短句
2. **输出**：纯文本播报台词，一键复制，直接粘贴至数字人工具生成短视频

## 模块 6：AI 播客生成模块

### 6.1 单人播客
- 调用通义千问生成 500 字左右播客文案
- 支持 SSE 流式输出，前端实时展示
- 无 AI Key 时降级为本地模板

### 6.2 双人对话播客
- 生成男女主持人（阿明/小雅）对话脚本，800-1000 字
- 自动标准化对话格式
- 段落级动画逐句展示

## 模块 7：TTS 语音合成模块

1. **单人模式**：整段文本合成一个音频
2. **双人对话模式**：逐段合成男女声，自动拼接播放队列
3. **前端播放器**：播放/暂停、进度条、时长显示、下载
4. **模型**：CosyVoice-V3-Flash / Qwen3-TTS-Flash（双模型自动降级）

## 模块 8：AI 诗词扩展模块

1. **输入**：当前花卉名称、朝代、代表诗句
2. **输出**：3-5 首相关诗词，含诗名、作者、朝代、经典句、30 字内赏析
3. **降级方案**：每花预置 3 首本地诗词，无 AI Key 时使用

# 七、AI 协同开发完整设计（作业核心要求）

## 7.1 AI 工具使用总表

```javascript
const express = require("express");
const cors = require("cors");
const { FlowerService } = require("./flowerService");

const app = express();
const flowerService = new FlowerService("flower_poetry_data.csv", "record.txt");

app.use(cors());
app.use(express.json());
app.use(express.static("frontend"));

app.get("/api/flowers/month/:month", (req, res) => {
  const data = flowerService.findByMonth(Number(req.params.month));
  if (!data) return res.status(400).json({ error: "无效月份" });
  res.json(data);
});

app.get("/api/recommend", (req, res) => {
  const results = flowerService.recommendByKeyword(req.query.keyword);
  res.json({ keyword: req.query.keyword, results });
});

app.get("/api/stats/dynasty", (_req, res) => {
  res.json(flowerService.getDynastyStats());
});

app.listen(8080, () => console.log("花月诗境服务已启动: http://localhost:8080"));
```
