# 一、引言

## 1\.1 编写目的

本文档完整定义《花月诗境》智能导览系统的产品需求、技术实现、数据规范、AI 协作流程、交付标准，统一开发、演示、报告撰写标准；同时满足课程作业硬性要求：可运行 Python 程序、数据处理、AI 工具应用、过程记录、讲解视频、运行截图，规范 AI 使用说明、留存提示词与人工修改记录。

## 1\.2 项目背景

2026 年央视春晚创意节目《贺花神》将十二月令花卉、古典诗词、历代文人意象结合，以数字化舞台形式传播传统月令花神文化。现有传播形式多为 PPT 图文资料展示，缺少交互式数据检索、智能匹配、自动化文化解析能力。
本项目以 Python 为基础，融合数据处理、AI 辅助创作、可视化统计，打造轻量化传统文化交互程序，实现**传统文化数字化、人机协同学习**，区别于纯文科资料展示类作业，充分体现计算思维、分支循环函数、数据结构、人工智能工具应用。

## 1\.3 术语定义

1. 十二花神：对应 1\-12 月月令花卉，匹配专属文人、经典诗词、朝代文化意象

2. Streamlit：Python 轻量化网页可视化框架，用于交互式界面展示

3. CSV：结构化数据表，存储十二花神全量基础数据

4. AI 协同：AI 仅作为初稿生成工具，所有内容人工核查、修正、定稿，非完全替代人工

5. 数字人播报：使用第三方 AI 数字人工具，导入程序生成的讲解台词生成短视频片段

6. 关键词匹配：基于字典映射、分支判断实现花卉智能推荐算法

1. 课程任务书（编程作品任务一提交规范）

# 二、产品概述与定位

## 2\.1 产品全称

《花月诗境：基于 Python 与 AI 的十二花神诗词智能导览系统》

## 2\.2 一句话定位

基于 Python 开发的交互式传统文化学习小程序，用户通过月份 / 情感关键词查询十二花神文化数据，系统自动输出花卉、诗人、朝代、经典诗句、文化意象、AI 人工修正讲解稿，附带朝代诗词数据可视化统计，支持生成数字人播报台词，完整覆盖编程作业全部考核要点。

## 2\.3 核心产品价值

1. **技术价值**：完整落地 Python 变量、分支、循环、自定义函数、CSV 数据读写、数据统计、可视化图表，满足程序设计考核要求；

2. **文化价值**：以春晚热点为切入点，系统化梳理十二月令花神与古典诗词体系；

3. **AI 实践价值**：标准化人机协同流程，完整记录 AI 生成、人工校验、修改迭代全过程，符合课程 AI 应用作业规范；

4. **展示价值**：网页交互界面 \+ 数据图表 \+ 数字人短片 \+ 完整演示视频，课堂展示效果优于控制台程序与静态 PPT。

## 2\.4 目标用户画像

1. 课程评审教师：考核 Python 编程能力、数据处理、AI 合理使用、项目完整性；

2. 学生使用者：传统文化爱好者，通过月份、情感关键词快速学习月令诗词；

3. 演示受众：课堂观众，可直观查看程序交互、数据分析、AI 讲解成果。

## 2\.5 产品核心差异化

摒弃传统纯图文 PPT 展示方案，不做高成本、难落地的全文生视频；采用**轻量化 Python 网页程序 \+ 本地 CSV 数据 \+ AI 辅助文案 \+ 低成本数字人短片**组合，低成本、易部署、代码可完整运行、技术亮点充足。

# 三、需求分析

## 3\.1 功能需求

### 基础核心功能（必做）

1. 十二花神按月查询功能：输入 / 选择月份，输出花卉、花神、诗人、朝代、诗句、文化意象、讲解稿；

2. 关键词智能推荐功能：输入情感 / 意象关键词，自动匹配对应花卉；

3. 朝代数据统计可视化：统计各朝代诗人数量，生成柱状统计图；

4. 查询结果本地保存：将用户查询记录导出至 TXT/CSV 文件；

5. 异常输入容错处理：非法月份、无效关键词给出友好提示。

### AI 辅助增值功能（作业加分项）

1. AI 生成标准化诗词讲解稿，人工修正后存入数据库；

2. 生成适配数字人朗读的简短播报台词；

3. 完整留存 AI 提示词、生成初稿、人工修改记录，用于报告撰写。

### 展示配套功能（可选）

1. 导出讲解文本，对接剪映 / 腾讯智影等工具生成数字人短视频；

2. 系统运行截图、录屏导出，用于提交材料。

## 3\.2 非功能需求

1. **易用性**：界面简洁，中文交互，无需专业编程基础即可操作；

2. **可运行性**：Windows/Mac 通用，仅需安装基础 Python 第三方库；

3. **可维护性**：数据独立存储于 CSV，新增花卉、诗句无需修改主代码；

4. **可展示性**：网页界面布局清晰，图表适配课堂投屏；

5. **可追溯性**：所有 AI 生成内容、修改过程、提示词完整留存，有据可查；

6. **轻量化**：无大型模型、无云端付费接口，本地运行，零成本。

## 3\.3 项目约束条件

1. 技术约束：仅使用 Python 基础语法、pandas、matplotlib、Streamlit，禁止复杂深度学习框架；

2. 成本约束：不使用付费文生视频、高算力 AI 工具，数字人采用免费在线工具；

3. 内容约束：诗词、诗人、朝代史料必须人工核验，禁止直接使用 AI 错误史料；

4. 交付约束：必须同时提交源代码、数据表、报告、视频、AI 截图、运行录屏；

5. 时间约束：整体开发周期适配课程作业周期，7 天内完成全流程。

## 3\.4 不包含功能（边界说明）

1. 不开发移动端 APP、小程序；

2. 不搭建后端服务器、数据库（仅本地 CSV 文件存储）；

3. 不实现实时在线大模型接口调用（讲解稿提前 AI 生成入库）；

4. 不做完整长篇数字人影片，仅 1 分钟精简播报片段；

5. 不拓展二十四节气、百花全量数据，仅聚焦十二花神核心数据。

# 六、核心功能模块详细说明

## 模块 1：十二花神按月查询模块

1. **输入**：用户下拉选择 1\-12 月 / 手动输入月份数字

2. **处理逻辑**：pandas 读取 CSV，匹配 month 字段，提取该行全部数据

3. **输出内容**：月份、花卉、花神、诗人朝代、经典诗句、文化意象、朝代诗词风格、AI 讲解文案

4. **异常处理**：输入数字 \> 12 或 \<1、中文月份文字错误，弹窗提示 “请输入 1\-12 有效月份”

## 模块 2：关键词智能花卉推荐模块

1. **内置关键词映射字典**

```Plain Text
keyword_map = {
    "高洁":["梅花","荷花","菊花"],
    "爱情":["桃花","杏花"],
    "隐逸":["菊花"],
    "团圆":["桂花"],
    "夏日":["荷花"]
}
```

2. **输入**：用户文本框输入单个意象关键词（高洁、思乡、春日、隐逸等）

3. **处理逻辑**：分支循环遍历字典，匹配关键词，返回对应花卉列表

4. **输出**：推荐花卉名称 \+ 对应月份简要提示

5. **容错**：无匹配关键词时提示 “暂无对应花卉，请更换关键词（高洁 / 爱情 / 隐逸 / 团圆 / 夏日）”

## 模块 3：朝代诗词数据统计可视化模块

1. **处理逻辑**：pandas 对 dynasty 字段分组计数，统计每个朝代对应花神数量

2. **可视化**：matplotlib 生成柱状图，横轴朝代、纵轴文人数量，中文适配字体

3. **输出**：网页内直接展示统计图，可截图保存用于作业材料

## 模块 4：查询结果保存模块

1. **触发方式**：用户点击 “保存本次查询记录” 按钮

2. **处理逻辑**：读取当前页面查询数据，追加写入 record\.txt 或导出单条 csv 记录

3. **用途**：留存交互记录，作为程序数据处理功能佐证

## 模块 5：数字人台词生成模块

1. **处理逻辑**：截取 ai\_script 字段，精简调整至 80 字以内短句

2. **输出**：纯文本播报台词，一键复制，直接粘贴至数字人工具生成短视频

# 八、运行部署与操作指南

## 8\.1 部署步骤

1. 将 \[app\.py\]\(app\.py\)、flower\_poetry\_data\.csv 放置于同一文件夹；

2. 打开终端 / CMD，执行依赖安装命令：
`pip install streamlit pandas matplotlib`

3. 启动程序命令：
`streamlit run app.py`

4. 自动弹出本地浏览器交互页面，即可全部功能操作。

## 8\.2 分步操作流程

1. 首页查看系统标题与功能介绍；

2. 功能一：下拉选择月份，点击查询，查看完整花神诗词信息；
