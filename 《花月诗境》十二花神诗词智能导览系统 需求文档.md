## 一、项目概述

### 1.1 项目背景

2026 年央视春晚《贺花神》节目将十二月令花卉、古典诗词、历代文人意象结合，成为文化热点。现有相关传播形式多为静态图文展示，缺乏交互式数据检索、智能匹配、自动化文化解析能力。本项目以 Node.js 为核心开发语言，融合数据处理、AI 辅助创作、可视化统计、语音合成能力，打造轻量化传统文化交互 Web 应用，既满足课程作业对编程能力、AI 应用的考核要求，又实现传统文化数字化传播。

### 1.2 系统定位

基于 Node.js + Express + 原生前端开发的轻量化交互式传统文化学习 Web 应用，面向课程评审教师、传统文化爱好者、课堂演示受众，支持通过月份 / 情感关键词查询十二花神文化数据，输出标准化文化解析内容，配套数据可视化、查询记录保存、播客文案生成、TTS 语音合成等能力，完整覆盖编程作业考核要点。

### 1.3 核心目标

1. 技术目标：落地变量、分支、循环、自定义函数、CSV 数据读写、数据统计、可视化等核心编程概念，满足程序设计课程考核要求；
2. 业务目标：实现十二花神文化数据的交互式查询、智能推荐、可视化分析，传播月令花神与古典诗词文化；
3. AI 应用目标：标准化人机协同流程，完整记录 AI 生成、人工校验、修改迭代全过程，符合课程 AI 应用作业规范；支持大模型实时调用生成播客文案与关键词匹配；
4. 展示目标：通过网页交互界面 + 数据图表 + 语音播客 + 数字人台词，实现优于控制台程序、静态 PPT 的课堂展示效果。

## 二、功能需求

### 2.1 核心基础功能（必实现）

#### 2.1.1 十二花神按月查询功能

|维度|具体要求|
|---|---|
|输入方式|下拉选择 1-12 月，或从首页月历卡片点击跳转|
|处理逻辑|后端通过 csv-parse 读取 CSV 数据表，匹配 "month" 字段，提取对应行全量数据|
|输出内容|月份、花卉名称、花神代表人物、所属朝代、经典诗句、文化意象、朝代诗词风格、AI 修正讲解稿、花卉图片、花神人物画像|
|异常处理|输入数字超出 1-12 范围时，返回 400 错误与提示信息|
|补充功能|花神人物、朝代名链接跳转百度百科，便于扩展学习|

#### 2.1.2 关键词智能花卉推荐功能

|维度|具体要求|
|---|---|
|输入方式|文本框输入情感/意象关键词，或点击预设关键词标签快速匹配|
|核心字典|内置关键词映射字典（38 个关键词），涵盖高洁、爱情、隐逸、团圆、夏日、清冷、孤傲、春日、温柔、明艳、华贵、浪漫、开拓、热烈、君子、清净、赤诚、向阳、忠直、内敛、清雅、自然、柔美、坚韧、风华、自持、仙雅、空灵等|
|处理逻辑|本地精确匹配 + 模糊匹配（字符级评分）双层策略；配置 AI Key 时优先使用大模型语义匹配|
|输出内容|推荐花卉名称 + 对应月份 + 花神人物 + 朝代 + 匹配理由|
|异常处理|无匹配关键词时，提示 "暂无对应花卉，请更换关键词"|

#### 2.1.3 朝代诗词数据统计可视化功能

|维度|具体要求|
|---|---|
|处理逻辑|后端对 CSV 表中 "dynasty" 字段分组计数，统计各朝代对应花神数量|
|可视化要求|基于 Chart.js 生成柱状图（横轴为朝代、纵轴为文人数量），渐变配色，圆角柱状，液态玻璃风格容器|
|输出形式|网页内直接展示统计图 + 数据明细表格，支持截图保存|

#### 2.1.4 查询结果保存功能

|维度|具体要求|
|---|---|
|触发方式|用户点击 "保存查询记录" 按钮|
|处理逻辑|读取当前页面查询数据，追加写入 record.txt 文件|
|记录格式|包含查询时间、查询类型、查询输入、查询结果，使用分隔线区分不同记录|
|功能价值|留存交互记录，佐证程序数据处理能力|

#### 2.1.5 异常输入容错功能

- 对非法月份（数字 > 12/<1）、无效关键词（无匹配结果）提供友好提示；
- 前端加载失败时展示明确的后端启动指引；
- API 请求失败有 Toast 提示，程序运行无崩溃、无闪退。

### 2.2 AI 辅助增值功能（加分项）

#### 2.2.1 AI 讲解稿生成与管理

- 基于十二花神基础数据，通过 AI 生成标准化讲解稿（80-120 字），经人工核查、修正后存入 CSV 数据表；
- 完整留存 AI 提示词、生成初稿、人工修改记录，用于作业报告撰写。

#### 2.2.2 数字人播报台词生成

- 截取 CSV 表中 "ai_script" 字段内容，精简调整为 80 字以内的口语化短句；
- 输出可一键复制的纯文本播报台词，支持直接粘贴至第三方数字人工具生成短视频片段。

#### 2.2.3 AI 播客文案生成（单人）

- 调用通义千问大模型（DashScope API），基于花神数据生成 500 字左右播客文案；
- 支持流式输出（SSE），前端逐字展示；
- 未配置 API Key 时自动降级为本地模板生成。

#### 2.2.4 AI 双人对话播客生成

- 生成男女主持人（阿明/小雅）双人对话形式的播客脚本，约 800-1000 字；
- 自动识别并标准化对话格式（男：/女：前缀）；
- 支持流式输出与段落级动画展示。

#### 2.2.5 TTS 语音合成

- 基于阿里云 DashScope CosyVoice / Qwen TTS 模型合成语音；
- 支持单人播报与男女对话双模式；
- 对话模式下逐段合成，自动拼接播放队列；
- 前端内置音频播放器，支持播放、暂停、进度显示、下载。

#### 2.2.6 AI 诗词扩展推荐

- 基于当前花卉查询更多相关诗词（3-5 首），含诗名、作者、朝代、经典句、简短赏析；
- 未配置 API Key 时使用本地内置诗词库（每花 3 首）。

#### 2.2.7 AI 语义关键词匹配

- 配置大模型时，支持语义级关键词匹配，不限于预设字典；
- 返回匹配理由，提升推荐的智能化与可解释性。

### 2.3 展示配套功能

- 液态玻璃（Glassmorphism）UI 设计，浅蓝诗境主题，适配课堂投屏；
- 12 月花卉图片 + 12 花神人物画像，丰富页面视觉；
- 平滑滚动、渐入动画、悬停动效，提升演示质感；
- 一键回到顶部，方便长页面浏览。

## 三、非功能需求

### 3.1 易用性

- 界面为中文交互，布局简洁，无专业编程基础的用户可独立操作；
- 功能按钮、输入框、输出区域位置合理，操作路径清晰（仅需 1-2 步即可完成核心查询）。

### 3.2 可运行性

- 兼容 Windows 10+/MacOS/Linux 操作系统；
- 仅依赖 Node.js 14+ 及 express、cors、csv-parse 基础库，安装部署流程简单；
- 提供 start.bat 一键启动脚本，自动检测依赖并安装。

### 3.3 可维护性

- 核心数据独立存储于 CSV 文件，新增/修改花卉、诗句、讲解稿无需修改主程序代码；
- 前后端分离架构，后端 RESTful API，前端模块化 JS；
- 代码函数命名规范，逻辑分层清晰，便于后期调试与迭代。

### 3.4 可展示性

- 网页界面适配课堂投屏分辨率，图表、文字清晰可辨；
- 支持快速切换功能模块，课堂演示流程流畅（单功能操作耗时≤10 秒）；
- 液态玻璃设计风格，视觉效果优于普通 Bootstrap 页面。

### 3.5 可追溯性

- 所有 AI 生成内容、修改过程、提示词完整留存（截图 + 文本记录）；
- 查询记录、数据统计结果可验证、可复现；
- 所有 API 有明确的请求/响应格式。

### 3.6 轻量化

- 核心功能（按月查询、关键词推荐、朝代统计、记录保存）无需 AI Key，全程本地运行；
- AI 功能为可选增强，配置 DASHSCOPE_API_KEY 即可启用；
- 无大型模型、无云端付费接口强依赖，零成本基础部署；
- 程序启动耗时≤3 秒，查询响应耗时≤0.5 秒。

## 四、数据需求

### 4.1 数据表结构

核心数据表名称：flower_poetry_data.csv，字段规范如下：

|字段名|字段说明|数据类型|示例|
|---|---|---|---|
|month|对应月份|数字|1, 3, 9|
|flower|月令花卉名称|文本|梅花、桃花、荷花|
|god_name|花神代表人物|文本|林逋、崔护、陶渊明|
|dynasty|人物所属朝代|文本|北宋、唐代、东晋|
|poem|代表经典诗句|文本|疏影横斜水清浅，暗香浮动月黄昏|
|keyword|匹配关键词（多词逗号分隔）|文本|高洁，清冷，孤傲|
|culture_image|文化意象解读|文本|寒梅临水、月下暗香，孤高隐逸、不染尘俗|
|dynasty_style|朝代诗词风格|文本|北宋诗文情理兼备，咏物常寄寓人格追求|
|ai_script|人工修正后的 AI 讲解稿|文本|80-120 字标准化讲解文案|

### 4.2 资源文件

|类别|路径|说明|数量|
|---|---|---|---|
|花卉图片|frontend/images/flowers/|每月月令花卉高清图|12 张|
|花神画像|frontend/images/gods/|每月花神代表人物画像|12 张|
|查询记录|record.txt|用户查询历史记录（运行后生成）|动态|

### 4.3 数据规范

- 诗词、诗人、朝代等史料信息需人工核验，确保无错误；
- AI 生成的讲解稿、播客文案需符合口语化、科普化要求，无晦涩文言；
- CSV 文件使用 UTF-8 编码，字段含逗号时用双引号包裹。

## 五、约束条件

### 5.1 技术约束

- 后端使用 Node.js + Express，前端使用原生 HTML/CSS/JS，可视化使用 Chart.js；
- AI 调用基于阿里云 DashScope API（通义千问 + CosyVoice）；
- 数据存储使用本地 CSV 文件，不搭建数据库服务。

### 5.2 成本约束

- 基础功能完全免费，本地运行零成本；
- AI 功能使用阿里云免费额度即可满足课程作业需求；
- 数字人制作采用免费在线工具（剪映/腾讯智影）。

### 5.3 内容约束

- 仅聚焦十二花神核心数据，不拓展二十四节气、百花全量数据；
- 所有文化内容需人工核查，禁止直接使用 AI 生成的错误史料。

### 5.4 交付约束

- 需同步提交源代码、CSV 数据表、AI 使用记录、运行截图、讲解视频、数字人播报片段；
- 整体开发周期需适配课程作业周期（7 天内完成全流程）。

## 六、验收标准

1. 程序可完整运行，无代码报错、无功能缺失，核心功能（按月查询、关键词推荐、统计可视化、记录保存）全部实现；
2. CSV 数据读取、统计、导出功能正常，异常输入可触发友好提示；
3. AI 使用记录（提示词、初稿、人工修改对比）完整可查，符合课程 AI 应用规范；
4. 可生成运行截图、录屏文件，数字人短片可正常播放；
5. 5-8 分钟讲解视频覆盖选题背景、代码演示、AI 应用过程、项目总结，播放正常；
6. （可选加分）播客生成、TTS 语音合成、AI 诗词推荐等增值功能可正常使用。

最终确定系统全称为《花月诗境：基于 Node.js 与 AI 的十二花神诗词智能导览系统》，简称 "花月诗境导览系统"。命名逻辑：

- "诗境" 突出诗词文化与沉浸式导览体验，体现系统的文化属性；
- 全称明确技术底座（Node.js + AI）与核心主题（十二花神诗词），符合课程作业的技术与文化双重考核要求，同时具备辨识度与文化美感。
